Three free tiers

What TradingView hands you for free

You do not need a data vendor or a charting startup. TradingView already ships the pieces. Here is the honest map of what you can self-host versus embed.

EMBED // ZERO SETUP

Widgets

Drop-in script tags. TradingView hosts them and supplies the data. Charts, heatmaps, screeners, calendars, news. The whole gallery below is live widgets.

APACHE 2.0 // SELF-HOST

Lightweight Charts

The open-source charting core on npm. Fully redistributable, you bring your own data. This is the real DIY TradingView. There is a live one below.

FREE // ACCESS-GATED

Advanced Charts

The full charting library: drawing tools, indicators, your own datafeed. Free but you apply for access and host it yourself. The pro tier of the DIY path.

YOURS // MCP

The AI layer

A screener plus an MCP server turns all of the above into something an agent can drive: screen, read a chart across timeframes, rank by factor. Pure TradingView.

The point

Now let your AI drive it

The screener ships an MCP server, so Claude can screen, compare, rank by factor, and read a chart across every timeframe at once. No swapping, no scraping. Pure TradingView data.

# claude_desktop_config.json
{
  "mcpServers": {
    "scanline": {
      "command": "/abs/path/.venv/bin/python",
      "args": ["/abs/path/run_mcp.py"]
    }
  }
}

# then just ask:
"Read NVDA across timeframes and rank
 mega-cap momentum, then chart the leader."

# analyze() answers, multi-timeframe:
NVDA 208.19: Neutral overall.
RSI 46 (bearish); MACD bullish;
timeframes mixed (1w/1m bull, 1h/4h/1d bear).

16 MCP tools and 4 canned prompts. The screening engine, plus symbol intelligence:

analyze chart read, all timeframes screen filters + factors + stats technical_rating the TV gauge compare symbols side by side chart deep link + widget sector_breakdown search_symbols run_factor_preset
